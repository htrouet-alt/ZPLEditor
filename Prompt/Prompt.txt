# ZPL Visual Editor — Detaylı Geliştirme Prompt'u

## Proje Özeti

Python ve PyQt6 kullanarak, mevcut ZPL (Zebra Programming Language) kodunu görsel olarak düzenleyebilen, tam özellikli bir 2D etiket tasarım editörü geliştir. Program iki panelden oluşacak: solda ZPL kod editörü, sağda WYSIWYG (What You See Is What You Get) görsel canvas. Kullanıcı mevcut bir ZPL kodunu yapıştırdığında, program bunu parse edip canvas üzerinde görsel elementlere dönüştürecek. Kullanıcı bu elementleri mouse ile seçebilecek, sürükleyebilecek, boyutlandırabilecek ve düzenleyebilecek. Her görsel değişiklik anında ZPL koduna yansıyacak ve her ZPL kodu değişikliği anında görsele yansıyacak (bidirectional sync).

---

## Teknoloji Stack

- **Dil:** Python 3.10+
- **GUI Framework:** PyQt6
- **Grafik Sistemi:** QGraphicsScene / QGraphicsView (zooming, panning, selection, transformation için)
- **ZPL Parsing:** Custom regex-based parser
- **Render Engine:** QPainter tabanlı custom rendering
- **Barcode Generation:** `python-barcode` kütüphanesi (1D barkodlar için), custom renderer (Code128, Code39, EAN13 vb.)
- **QR Code:** `qrcode` kütüphanesi
- **Font Handling:** Zebra yerleşik fontları için bitmap font mapping + TrueType font desteği

### 2. ZPL Kod Editörü (`code_editor.py`)

`QPlainTextEdit` tabanlı, ZPL syntax highlighting destekli bir kod editörü.

**Syntax Highlighting Kuralları:**
- `^XA`, `^XZ` — Etiket başlangıç/bitiş → Koyu mavi, bold
- `^FO` — Alan orijini → Yeşil
- `^FD` — Alan verisi → Turuncu
- `^A` — Font seçimi → Mor
- `^BC`, `^B3`, `^BY`, `^BQ` — Barkod komutları → Kırmızı
- `^GB`, `^GC`, `^GD` — Grafik komutları → Turkuaz
- `^CF` — Varsayılan font → Mor
- `^CI` — Karakter seti → Gri
- `^PQ` — Yazdırma miktarı → Gri
- Sayılar → Açık mavi
- Virgüller (parametre ayırıcı) → Koyu gri

**Özellikler:**
- Satır numaraları
- Mevcut satır vurgulama
- Auto-completion (temel ZPL komutları için) — `^` yazınca komut listesi açılır
- Bracket matching (`^` ... `,` grupları)
- Gerçek zamanlı sync: Kod değiştiğinde 500ms debounce ile canvas güncellenir
- Hata işaretleme: Parse edilemeyen satırlar kırmızı arka planla vurgulanır
- Kod katlama (^XA...^XZ blokları)

**Bidirectional Sync Mekanizması:**
- Canvas'ta bir element değiştiğinde, ZPL kodu otomatik güncellenir. Güncelleme sırasında editördeki imleç pozisyonu ve seçim korunur.
- Editörde kod değiştiğinde, 500ms debounce sonrası parser çalışır ve canvas güncellenir. Değişmeyen elementlerin canvas pozisyonları ve seçim durumları korunur.
- Sync sırasında sonsuz döngüyü önlemek için bir `_syncing` flag kullanılır.

### 3. ZPL Parser (`zpl_parser.py`)

ZPL kodunu parse edip element listesine dönüştüren modül. Bu modül projenin en kritik parçasıdır.

**Desteklenmesi Gereken ZPL Komutları (Öncelik Sırasıyla):**

**Öncelik 1 — Temel (Mutlaka):**
- `^XA` / `^XZ` — Etiket başlangıç/bitiş
- `^FO x,y` — Alan orijini (element pozisyonu)
- `^FD data ^FS` — Alan verisi ve alan sonu
- `^A font,orientation,height,width` — Font seçimi
- `^CF font,height,width` — Varsayılan font
- `^GB width,height,thickness,color,rounding` — Grafik kutu (çizgi ve dikdörtgen)
- `^FT x,y` — Alan tipi (baseline konum)
- `^CI charset` — Karakter set tanımı

**Öncelik 2 — Barkod ve Grafik:**
- `^BC orientation,height,interpretation,above,checkdigit` — Code 128 barkod
- `^B3 orientation,checkdigit,height,interpretation,above` — Code 39 barkod
- `^BY module_width,ratio,height` — Barkod varsayılanları
- `^BQ orientation,model,magnification` — QR kod
- `^GC diameter,thickness,color` — Grafik daire
- `^GD width,height,thickness,color,orientation` — Grafik çapraz çizgi

**Öncelik 3 — Gelişmiş:**
- `^FB width,max_lines,line_spacing,alignment,hanging` — Alan bloğu (çok satırlı metin)
- `^FW orientation` — Alan yönü
- `^GFA format,total_bytes,row_bytes,rows,data` — Grafik alan (bitmap)
- `^PW width` — Yazdırma genişliği
- `^LL length` — Etiket uzunluğu
- `^LH x,y` — Etiket home pozisyonu
- `^FX comment` — Yorum satırı (göz ardı edilir)

**Parser Algoritması:**

```python
def parse_zpl(zpl_code: str) -> LabelModel:
    """
    ZPL kodunu parse eder ve LabelModel döndürür.
    
    Algoritma:
    1. ^XA ile ^XZ arasındaki kodu al
    2. Kodu '^' karakterinden böl (her biri bir komut)
    3. Her komutu komut_kodu + parametreler olarak ayır
    4. State machine ile elementleri oluştur:
       - ^FO görüldüğünde yeni element başlat, pozisyonu kaydet
       - Sonraki komutlar (^A, ^FD, ^GB vb.) bu elemente özellik ekler
       - ^FS görüldüğünde elementi tamamla
    5. Global komutları (^PW, ^LL, ^CF, ^CI) etiket modeline uygula
    """
```

**Önemli Parse Kuralları:**
- Parametreler virgülle ayrılır ve boş bırakılabilir (varsayılan değer kullanılır): `^A0N,30,` → font=0, orientation=N, height=30, width=default
- `^FO` her zaman yeni bir elementin başlangıcıdır
- Bir `^FO` ile sonraki `^FS` arasındaki tüm komutlar aynı elemente aittir
- `^FD` her zaman verinin kendisini içerir, `^FS` ile sonlanır
- Birden fazla `^FD` olabilir (ender, ama mümkün)
- `^GB` hem çizgi hem dikdörtgen olabilir: eğer width veya height 0 ise çizgidir
- Koordinatlar dot cinsindendir (203 DPI'da 1 dot ≈ 0.125mm)

### 4. ZPL Generator (`zpl_generator.py`)

Element listesinden ZPL kodu üretir. Parser'ın tersidir.

**Kurallar:**
- Her element için önce `^FO x,y` üret
- Elementin tipine göre ilgili komutları üret
- Metin elementi: `^A font,orientation,height,width ^FD text ^FS`
- Kutu/çizgi: `^GB width,height,thickness,color,rounding ^FS`
- Barkod: `^BY ... ^BC ... ^FD data ^FS`
- Sonucu `^XA ... ^XZ` ile sarla
- Global ayarları (^PW, ^LL, ^CF, ^CI, ^LH) etiketin başına ekle
- Elementleri Y koordinatına göre sırala (yukarıdan aşağıya)
- Okunabilirlik için her elementi yeni satıra yaz
- Yorum olarak element tipini ekle: `^FX -- Text Element --`

### 6. Grafik Elementler (`elements/`)

Her element `QGraphicsItem` tabanlıdır ve aşağıdaki ortak özelliklere sahiptir:

**Base Element (`base_element.py`):**

```python
class BaseElement(QGraphicsItem):
    """
    Tüm ZPL elementlerinin base class'ı.
    
    Ortak özellikler:
    - Pozisyon (x, y) — dot cinsinden
    - Boyut (width, height) — dot cinsinden
    - Seçilebilir, sürüklenebilir, boyutlandırılabilir
    - 8 resize handle (4 köşe + 4 kenar ortası)
    - Seçildiğinde mavi noktalı çerçeve
    - Hover'da açık mavi vurgulama
    - Sağ tık context menü
    - Sürükleme sırasında snap-to-grid
    - Sürükleme sırasında hizalama çizgileri (diğer elementlere göre)
    """
    
    # Resize handle'lar:
    # [TL]----[T]----[TR]
    #  |               |
    # [L]             [R]
    #  |               |
    # [BL]----[B]----[BR]
    #
    # Her handle 8x8 piksel mavi kare
    # Köşe handle'lar: her iki yönde boyutlandırır
    # Kenar handle'lar: sadece ilgili yönde boyutlandırır
```

**Metin Elementi (`text_element.py`):**
- ZPL komutları: `^FO`, `^A`, `^FD`, `^FS`
- Özellikler: font, font_size, rotation (0, 90, 180, 270), text_content
- Çift tıklayınca inline editing modu: doğrudan canvas üzerinde metin düzenleme
- Font mapping: ZPL font ID (0-9, A-Z) → yaklaşık QFont karşılığı
- Font 0 (varsayılan) → monospace benzeri font
- Resize: font boyutunu değiştirir (genişlik ve yükseklik)

**Dikdörtgen/Çizgi Elementi (`line_element.py`, `box_element.py`):**
- ZPL komutları: `^FO`, `^GB width,height,thickness,color,rounding`
- Özellikler: width, height, border_thickness, color (B/W), corner_rounding
- Eğer width veya height ≤ thickness ise: çizgi olarak render et
- Eğer rounding > 0 ise: köşeleri yuvarlat
- Resize: width ve height'ı değiştirir
- Thickness bir özellik panelinden ayarlanır

**Daire Elementi (`circle_element.py`):**
- ZPL komutu: `^FO`, `^GC diameter,thickness,color`
- Özellikler: diameter, border_thickness, color
- Resize: diameter'ı değiştirir (aspect ratio kilitli)

**Çapraz Çizgi (`diagonal_line.py`):**
- ZPL komutu: `^FO`, `^GD width,height,thickness,color,orientation`
- Özellikler: width, height, thickness, color, orientation (R/L)
- R = sağ eğimli (\), L = sol eğimli (/)

**Barkod Elementi (`barcode_element.py`):**
- ZPL komutları: `^FO`, `^BY`, `^BC`/`^B3` vb., `^FD`, `^FS`
- Özellikler: barcode_type, data, module_width, height, show_interpretation
- Canvas üzerinde gerçek barkod render edilir (python-barcode ile)
- Resize: yükseklik ve genişlik ayarlanabilir
- Çift tıklayınca barkod verisini düzenleme dialog'u açılır

**QR Kod Elementi (`qr_element.py`):**
- ZPL komutu: `^FO`, `^BQ`, `^FD`, `^FS`
- Özellikler: data, magnification, error_correction
- Canvas üzerinde gerçek QR kod render edilir (qrcode kütüphanesi ile)
- Resize: magnification'ı değiştirir

### 12. Labelary API Entegrasyonu (Opsiyonel ama Önerilir)

Labelary.com'un API'si ile ZPL kodunun gerçek barkod yazıcı çıktısını preview olarak gösterme imkanı:

```python
import requests

def get_labelary_preview(zpl_code: str, dpi: int = 203, 
                          width: float = 4, height: float = 6) -> QPixmap:
    """
    Labelary API üzerinden ZPL kodunun gerçek render'ını al.
    
    Endpoint: http://api.labelary.com/v1/printers/{dpi}dpi/labels/{width}x{height}/0/
    Method: POST
    Body: ZPL kodu (plain text)
    Response: PNG image
    """
    url = f"http://api.labelary.com/v1/printers/{dpi}dpi/labels/{width}x{height}/0/"
    response = requests.post(url, data=zpl_code.encode('utf-8'),
                              headers={'Accept': 'image/png'})
    pixmap = QPixmap()
    pixmap.loadFromData(response.content)
    return pixmap
```

Bu preview, canvas'ın üstünde veya ayrı bir sekmede "Yazıcı Önizleme" olarak gösterilebilir. Böylece kullanıcı kendi render'ı ile gerçek çıktıyı karşılaştırabilir.
